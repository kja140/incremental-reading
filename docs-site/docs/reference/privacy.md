# Privacy and permissions

Incremental Reading Toolkit is local-first. It has no telemetry, advertisements, account requirement, or network-based update mechanism.

| Capability | Behavior |
|---|---|
| Clipboard | Read only when you run a clipboard capture command. |
| External PDFs | Read only from the local path stored on the active source; the PDF is not modified. |
| Vault | Managed content is read and written within configured Toolkit folders. |
| Network | The plugin itself makes no requests. Opening a YouTube embed allows Obsidian to contact YouTube. |
| Migrations | Legacy-card and date migrations require explicit commands. |

The examples and artwork in this documentation are synthetic and generic. They do not use personal vault notes, books, or machine data.
