# Elements and the knowledge tree

The Toolkit treats your material as a hierarchy of elements:

```text
Category
└── Source
    ├── Extract
    │   └── Extract
    └── Extract
```

A **category** organizes related work. A **source** points to the original material. An **extract** is a selected passage that deserves its own return visits. Extracts can have children, so an idea can be refined progressively.

Parent relationships are stored in note frontmatter and visualized in the Knowledge Tree. You can drag an element to reparent it, move the active element with a command, postpone a whole subtree, or open the parent from a child.

The tree is organizational, not a requirement for every note. Start with a source and a few extracts; introduce categories when the collection becomes hard to scan.
